システム構造・技術スタック・カスタマイズ方法

🏗️ システム概要

選択支援メニューシステムは、HTMLとJavaScriptのみで構成された軽量なWebアプリケーションです。サーバー不要で動作し、静的ファイル配信のみで運用できます。

🔧 カスタマイズ方法

1. メニュー画像を差し替える

menu_data.js の各アイテムの image プロパティを変更します。

// menu_data.js window.MENU_ITEMS = [ { id: 'shoyu_ramen', name: '醤油ラーメン', image: 'https://...', // ← ここを変更 ... }, ... ];
💡 画像の用意方法：
• 施設で撮影した写真をアップロードし、そのURLを使う
• AI画像生成サービス（Meshy.ai等）で作成
• フリー素材サイト（Unsplash, Pexels等）から選ぶ
• 画像URLが http:// の場合は https:// が必要（HTTPS必須）

2. メニューを追加する

menu_data.js の配列に新しいオブジェクトを追加します。

{ id: 'original_menu1', // 一意のID name: '当施設特製○○', // 表示名 reading: 'とくせい○○', // ふりがな romaji: 'Tokusei XX', // ローマ字（任意） category: '和食', // カテゴリ emoji: '🍱', // 絵文字 image: 'https://...', // 画像URL tags: ['やさしい', 'あたたかい'] // タグ }

3. カテゴリを追加する

menu_data.js 末尾の MENU_CATEGORIES に追加します。

{ id: 'スイーツ', label: 'スイーツ', emoji: '🍰', color: '#E8789F' }

4. ARで使う料理を変更する

WebAR本番.html 内の FOOD_MENU 配列を編集します。

const FOOD_MENU = [ { id: 'ramen', name: 'ラーメン', markerType: 'preset', // 'preset' または 'barcode' markerPreset: 'hiro', // Hiroマーカー使用 color: 0xE8A53F, image: 'https://...' }, { id: 'curry', name: 'カレーライス', markerType: 'barcode', markerValue: 10, // バーコード番号 color: 0xB8722F, image: 'https://...' } ]
⚠️ マーカーの制約：
• preset は hiro のみ動作確認済み（kanji は画像読込に失敗）
• barcode は 0〜63 の範囲で指定可能
• 最大3〜5マーカーまでが実用範囲（それ以上は認識が不安定）

5. 色・デザインを変更する

各HTMLファイルの <style> セクションを編集します。主要カラー：

カラー変数 色コード 用途
メインブランド #C85A2B ボタン・強調色（オレンジ）
ダークブラウン #3D2E1F 文字色・ヘッダー
アクセント #FFD4A8 ハイライト・背景
ベージュ #FFF8F0 ページ背景
成功色 #4CD964 OK・チェック

🎯 マーカーの技術仕様

現在使用しているマーカー

食べ物 マーカー種別 識別子
🍜 ラーメン Preset (Hiro) preset="hiro"
🍛 カレーライス Barcode type="barcode" value="10"
🍣 おすし Barcode type="barcode" value="5"

バーコードマーカーの仕組み

AR.jsのバーコードマーカーは、3×3グリッドの白黒パターンで表現されます。0〜63までの値（6ビット）を持つことができます。

// バーコードマトリックスの構造 // [0,0,0] ← 行0 // [0,0,0] ← 行1 // [1,0,1] ← 行2 = 値5 // 検出設定（WebAR本番.html内） arjs="sourceType: webcam; detectionMode: mono_and_matrix; matrixCodeType: 3x3;"
💡 マーカー追加の方法：新しい食べ物を追加する場合は、markerValue を他の値（例：20, 25, 30）に設定します。同じ値を複数の料理に割り当てないよう注意してください。

💾 データ永続化

このシステムは localStorage を使用してデータを保存します。サーバーは使用しません。

キー 保存内容 用途
senshakuSelectedIds 選んだメニューID 配列 メイン画面の選択状態
senshakuPanel 選択中のタブID（p1〜p4） 4案の表示状態
webar_camera_facing "environment" または "user" カメラ向き設定
webar_autostart（SessionStorage） カメラ切替時の自動起動フラグ 再起動時の動作
⚠️ 注意：localStorageはブラウザ単位で保存されます。別の端末やシークレットモードでは設定が引き継がれません。
